## ステップ 5 — Phase 4: 眼内媒質の置換（ガス／シリコンオイル／無水晶体）✅ 完了（ユーザー確認済み）

ボタンで切替えられるようにし、水晶体の有無も独立してトグル可能。

教育意図（spec 通り）:
- 媒質変化は水晶体後面の屈折率差として現れる → 水晶体を 1 枚に集約した模型では再現不可
- シリコンオイル充填 → 水晶体後面での屈折力減少 → 遠視化（模型眼で約 +8D）
- ガス充填 → 屈折率差が逆向きに増加 → 近視化（オイルと逆方向、強い）
- 無水晶体眼 → 全体の屈折系が大きく弱体化 → 強い遠視化
- 無水晶体＋シリコンオイル → 角膜が高屈折率液面を作るため逆に近視化（教育的に重要）

## ステップ 6 — Phase 5: PSF による見え方シミュレーション

瞳孔関数 → PSF（点像分布）→ 物体（C字・E字・点光源）との畳み込みで「網膜にどう映るか」を表示。

教育意図:
- デフォーカスや乱視が「見え方」にどう現れるかをビジュアル化（数値から体験へ）
- 瞳径を変えると焦点深度が変わる体験
- 乱視軸が変わると C 字の見え方が方向依存になる体験
- 視力 0.1 → 2.0 のサイズ変化で「どこまで判別できるか」の臨床的境界を示す

UI 仕様（ユーザー確定）:
- 眼の状態: Phase 5 専用スライダで独立に調整
  - 球面度数 [D]（例 −6.0〜+6.0）
  - 円柱度数 [D]（例 −4.0〜0.0）
  - 軸角度 [°]（0〜180）
  - 瞳径 [mm]（1〜7）
- 視標: 視距離スライダ＋視標サイズ（視力値）スライダ 両方
- 視標タイプ切替: ランドルト環 / 点光源（PSF直接表示） / 文字列（E字・かな・数字）

実装メモ:
- 既存 `src/optics/psf.ts` を活用（あれば中身を確認のうえ）
- 瞳孔関数を 128×128 程度でサンプリング → FFT で PSF → 視標との畳み込み
- 表示はキャンバス（Three.js を使わず Canvas2D で十分かを検討）
- パフォーマンス: スライダ操作時にデバウンス／前計算

### 2026-06-11 Phase 4 視距離・視力値スライダの修正

- 報告内容: 「視力値と物理距離のつまみを動かしても PSF 像が全く変化しない」
- 根本原因:
  - `distanceM` が読み出し（物理サイズ表示）にのみ使われており、PSF 計算に渡されていなかった
  - 視力値（acuityDecimal）は視標サイズに影響するが、ユーザーは点光源モードで観察していた
    （点光源モードは原理上 PSF そのものを表示するので視標サイズは効かない）
- 修正内容:
  - `src/viz/phase4.ts`: 無調節モデルとして `effectiveSphereD = sphereD + 1/distanceM` を導入し、
    PSF 計算にこの実効値を渡すよう変更
  - 読み出しを「等価球面（屈折異常）／調節要求／実効デフォーカス」の3行に分け、
    球面度数と視距離の相互作用が数値で見えるようにした
  - `src/viz/controlsPhase4.ts`: 視距離スライダにヒント追加、
    視力値ヒントに「点光源モードでは効果なし」を明記、操作ヒントに視距離効果を追記
- 検証: `npx tsc --noEmit` 通過
- 学び: `tasks/lessons.md` に Phase 4 同パターン再発として記録
